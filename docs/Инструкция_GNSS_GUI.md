# Инструкция пользователя — GNSS Processing GUI v3.5.6

## Назначение программы

**GNSS Processing GUI** — настольная программа для импорта, визуализации и обработки временных рядов GNSS/GPS, связанных прежде всего с высотой. Программа позволяет загрузить файл наблюдений, проверить и нормализовать временную шкалу, удалить выбросы, применить фильтр Butterworth и медианный фильтр, построить график и сохранить результаты обработки. [file:753][file:748]

Поддерживаются входные файлы форматов `.csv`, `.txt`, `.tsv`, `.xls`, `.xlsx`, `.pos`. Тестовые и входные данные по структуре проекта обычно находятся в каталоге `data/input/`, а результаты работы сохраняются в `data/output/`. [file:753][file:748]

## Запуск программы

Если используется готовая Windows-сборка, откройте папку программы и запустите `GNSS_GUI.exe`. Для one-folder сборки файл `.exe` должен находиться рядом с папкой `_internal`, потому что часть библиотек PySide6 и других зависимостей загружается именно оттуда. [file:758][file:753]

Если программа запускается из исходного кода, используйте одну из команд в корне проекта:

```bash
python run_gui.py
```

или

```bash
python packaging/run_gnss_gui.py
```

После запуска откроется главное окно: слева расположены параметры импорта и фильтрации, в центре — кнопки действий и график, справа — журнал обработки. В верхней части центральной панели отображается текущий статус выполнения операций. [file:753]

## Быстрый старт

Рекомендуемая последовательность работы:

1. Нажмите **Обзор / Browse** и выберите файл для обработки.
2. При необходимости вручную укажите колонки времени и высоты.
3. Нажмите **Load** для загрузки данных.
4. Нажмите **Validate time** для проверки и нормализации времени.
5. Нажмите **Outlier** для удаления выбросов.
6. При необходимости примените **Butterworth**.
7. При необходимости примените **Median**.
8. Нажмите **Plot**, чтобы обновить отображение.
9. Сохраните результат через **Save CSV**, **Save JPG** или **Save Bundle**. [file:753]

## Загрузка данных

Для выбора входного файла нажмите кнопку обзора и укажите нужный файл. По умолчанию программа открывает диалог выбора в папке `data/input/`, если такая папка существует в проекте. [file:753]

Программа умеет автоматически определять колонки времени и высоты. Для времени она ищет варианты вроде `utc_time`, `datetime_utc`, `Дата-Время`, а для высоты — `height`, `height_m`, `Высота (м)`. Если автоматическое распознавание не сработало, впишите имена колонок вручную в соответствующие поля перед загрузкой. [file:753]

После успешного импорта в журнале отображается информация о количестве строк и выбранных колонках, а загруженные данные становятся доступны для построения графика и последующей обработки. [file:753]

## Этапы обработки

### 1. Проверка времени

Кнопка **Validate time** выполняет проверку и нормализацию временной колонки. После этого формируется таблица валидированных данных, которая используется на следующих этапах обработки. [file:753]

### 2. Удаление выбросов

Кнопка **Outlier** запускает sigma-фильтрацию и очистку аномальных точек. Основные параметры этого шага:

- `k sigma` — порог выбросов;
- `window sec` — длина окна в секундах;
- `min group` — минимальный размер группы;
- `max group epochs` — максимальный размер группы. [file:753][file:748]

После выполнения шага программа формирует очищенный набор данных, таблицу удалённых точек и информацию о флагах качества. [file:753]

### 3. Фильтр Butterworth

Кнопка **Butterworth** применяет сглаживание к текущему рабочему ряду. Используются параметры:

- `order` — порядок фильтра;
- `period min` — период в минутах. [file:753][file:748]

Результат записывается в колонку `heightbw`, а также может использоваться как основа для следующего этапа — медианного фильтра. [file:753][file:748]

### 4. Медианный фильтр

Кнопка **Median** выполняет медианную фильтрацию по остаткам Butterworth. Настраиваются параметры:

- `window points` — размер окна в точках;
- `threshold mm` — порог в миллиметрах. [file:753][file:748]

Результат включает колонки `heightmed`, `residualmedmm`, а также признак медианного выброса. Если включён режим live-пересчёта, медианный фильтр обновляется автоматически после изменения параметров или после пересчёта Butterworth. [file:753]

## Работа с графиком

На графике можно включать и выключать отдельные слои:

- `Raw` — исходные данные;
- `Validated` — данные после проверки времени;
- `Clean` — данные после удаления выбросов;
- `BW` — результат фильтра Butterworth;
- `Med` — результат медианного фильтра. [file:753]

Для оси Y доступны два режима:

- абсолютная высота в метрах;
- отклонение от среднего в миллиметрах. [file:753]

Для оси X доступны режимы:

- весь диапазон;
- последние 1 сутки;
- последние 7 суток;
- ручной диапазон через поля **From** и **To**. [file:753]

## Сохранение результатов

Программа поддерживает несколько вариантов экспорта:

- **Save CSV** — сохранить текущий отображаемый набор данных в CSV; [file:753]
- **Save JPG** — сохранить текущий график в формате JPG; [file:753]
- **Save Bundle** — сохранить комплект результатов: CSV, JPG и текстовый лог обработки. [file:753]

По умолчанию результаты сохраняются в подкаталоги папки `data/output/`:

```text
data/output/csv/
data/output/jpg/
data/output/logs/
data/output/final export/
``` [file:753]

Программа также позволяет экспортировать и импортировать JSON-файлы с параметрами фильтрации, чтобы повторно использовать одинаковые настройки обработки. [file:753]

## Типичные замечания

- Для корректной обработки в файле должна присутствовать колонка времени и колонка высоты. [file:753]
- Если автоматическое определение колонок сработало неверно, задайте их вручную перед загрузкой. [file:753]
- Если используется готовая Windows-сборка, скачивайте её из раздела **GitHub Releases**, а не из содержимого репозитория. Готовые ZIP- и EXE-сборки предназначены для публикации через Releases. [web:1022][web:1034]
- Если `.exe` находится в one-folder сборке, не переносите его отдельно от папки `_internal`, иначе программа может не запуститься. [file:758]
